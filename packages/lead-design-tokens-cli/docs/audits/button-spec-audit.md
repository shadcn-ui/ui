# Button spec audit

> Read-only audit comparing the Lead `<Button>` React API to its Figma
> source surface. Generated by `npm run audit:button-spec`. Per JES-77,
> this prototype is Button-only and read-only — no Figma API calls, no
> production code changes, no automated code-change PRs.

## Mapped (5)

React surface that has a corresponding Figma source mapping.

- `variant`
- `size`
- `disabled`
- `loading`
- `children ← label (Figma TEXT: "Button Text")`

## Unmapped React props (3)

React props on `<Button>` with no corresponding Figma surface today.

- `loadingLabel`
- `leadingIcon`
- `trailingIcon`

Plus `HTMLButtonAttributes` pass-through (e.g. `type`, `aria-*`, `onClick`) — not enumerated here.

## Unmapped Figma props / naming drift (10)

Figma values documented in the `Button.figma.tsx` header comment but not present in the `props: { ... }` mapping. Exact-name comparison only; no fuzzy matching.

- **Variant:** `Link`
- **Size:** `icon`
- **State:** `Default`, `Hover`, `Focus`, `Pressed`
- **Other (explicitly not mapped in pilot):** `Show Left Icon`, `Left Icon`, `Show Right Icon`, `Right Icon`

_Note: `State=Default` in Figma is the implicit baseline (no `disabled` and no `loading`); it's listed above for completeness but has an implicit React mapping via the default false values._
