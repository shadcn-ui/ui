# 5-Day Launch Plan

## Day 1: Polish & Test

### Morning (2-3 hours)
- [ ] Run `pnpm install` and `pnpm dev` — verify everything works
- [ ] Test all pages and navigation
- [ ] Test dark mode toggle
- [ ] Test responsive design on mobile
- [ ] Fix any visual bugs

### Afternoon (2-3 hours)
- [ ] Review code quality — remove any TODOs or placeholder comments
- [ ] Ensure consistent naming conventions
- [ ] Verify all imports work correctly
- [ ] Run build: `pnpm build` — fix any errors

### Evening (1 hour)
- [ ] Create a simple demo video (Loom, 2-3 minutes max)
- [ ] Or: Record GIF of dashboard in action

---

## Day 2: Screenshots & Assets

### Morning (2 hours)
- [ ] Take all screenshots (follow SCREENSHOTS.md checklist)
- [ ] Dashboard (light mode) — this is your hero image
- [ ] Dashboard (dark mode)
- [ ] Login page
- [ ] Settings page
- [ ] Billing page
- [ ] Analytics page
- [ ] Empty state (Customers page)
- [ ] Mobile view

### Afternoon (2 hours)
- [ ] Edit screenshots in Figma/Canva:
  - Add device frames (optional)
  - Create hero image with text overlay
  - Ensure consistent sizing
- [ ] Create thumbnail (1280x720 or 1920x1080)

### Evening (1 hour)
- [ ] Create ZIP file of the product
- [ ] Test ZIP: download, unzip, run — confirm it works
- [ ] Final file size check (should be under 10MB)

---

## Day 3: Gumroad Setup

### Morning (2 hours)
- [ ] Create Gumroad account (if needed)
- [ ] Set up Stripe/PayPal for payouts
- [ ] Create new product
- [ ] Upload product ZIP file
- [ ] Add all screenshots

### Afternoon (2 hours)
- [ ] Write product description (use GUMROAD-COPY.md)
- [ ] Set price: $29 (launch special)
- [ ] Configure:
  - [ ] Product type: Digital product
  - [ ] License key: Not needed
  - [ ] Limit quantity: No
  - [ ] Allow variants: No (for now)
- [ ] Add tags for discoverability

### Evening (1 hour)
- [ ] Preview product page
- [ ] Test purchase flow (use test mode or buy yourself)
- [ ] Verify download works after purchase
- [ ] Set product to "Published" but not yet promoted

---

## Day 4: Pre-Launch Marketing

### Morning (2 hours)
- [ ] Write launch tweet/thread:
  ```
  I built a SaaS dashboard starter kit.

  - Login, Dashboard, Settings, Billing pages
  - React + TypeScript + Tailwind
  - Dark mode, responsive
  - $29 launch price

  [Screenshot]

  Link in bio (or direct link)
  ```
- [ ] Prepare 3-5 follow-up tweets showing features

### Afternoon (2 hours)
- [ ] Find relevant communities:
  - [ ] r/SideProject
  - [ ] r/webdev (Show Off Saturday)
  - [ ] Indie Hackers
  - [ ] Twitter (hashtags: #buildinpublic, #indiehackers)
  - [ ] Dev.to (write a short article)
- [ ] Draft posts for each (slightly different angles)

### Evening (1 hour)
- [ ] Schedule tweets (or set reminders)
- [ ] Prepare launch day message for friends/network
- [ ] DM 3-5 people who might be interested for early feedback

## Emergency Checklist

If something goes wrong:

### Product Issues
- [ ] Test the ZIP file yourself
- [ ] Re-upload if corrupted
- [ ] Respond to buyers immediately

### No Sales
- [ ] Check product page is public
- [ ] Verify payment works
- [ ] Ask for feedback on messaging
- [ ] Consider lowering launch price

### Negative Feedback
- [ ] Respond professionally
- [ ] Fix legitimate issues quickly
- [ ] Don't argue — improve the product
