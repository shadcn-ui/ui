Force UI works with the official `shadcn` CLI. Use a Force UI preset URL to install the style, theme, and font, then use the registry to add components.

These steps are written to be copy-pasteable for both humans and coding agents.

## What installs what

- `shadcn init` with a Force UI preset installs the base setup for a new project.
- `shadcn apply` with a Force UI preset switches an existing project to Force UI.
- `shadcn add` installs Force UI component source files.
- Custom variants ship with the component source. There is no separate variant install step.

## Hosted URLs

```bash
export FORCE_UI_REGISTRY="https://shadcn-force-ui.vercel.app/r"
export FORCE_UI_PRESET_RADIX="https://shadcn-force-ui.vercel.app/init?base=radix&style=force-ui&baseColor=force-ui&theme=force-ui&chartColor=force-ui&iconLibrary=lucide&font=noto-sans&rtl=false&menuAccent=subtle&menuColor=default&radius=default"
export FORCE_UI_PRESET_BASE="https://shadcn-force-ui.vercel.app/init?base=base&style=force-ui&baseColor=force-ui&theme=force-ui&chartColor=force-ui&iconLibrary=lucide&font=noto-sans&rtl=false&menuAccent=subtle&menuColor=default&radius=default"
```

## New React project

### Radix UI

```bash
REGISTRY_URL="$FORCE_UI_REGISTRY" \
npx shadcn@latest init -t vite -b radix -p "$FORCE_UI_PRESET_RADIX" -y
```

### Base UI

```bash
REGISTRY_URL="$FORCE_UI_REGISTRY" \
npx shadcn@latest init -t vite -b base -p "$FORCE_UI_PRESET_BASE" -y
```

After init, React projects should have one of these style values in `components.json`:

- `radix-force-ui`
- `base-force-ui`

Do not set the React style to `force-ui`. The hosted React registry paths use `radix-force-ui` and `base-force-ui`.

## Existing React project

### Switch the full project to Force UI

```bash
REGISTRY_URL="$FORCE_UI_REGISTRY" \
npx shadcn@latest apply --preset "$FORCE_UI_PRESET_RADIX" -y
```

Use the Base preset URL instead when your project uses Base UI.

### Update only the theme and font

```bash
REGISTRY_URL="$FORCE_UI_REGISTRY" \
npx shadcn@latest apply --preset "$FORCE_UI_PRESET_RADIX" --only theme,font -y
```

This updates the CSS variables and font setup without reinstalling UI components.

## Configure the registry namespace

After `init` or `apply`, add an explicit registry mapping to `components.json`. The official CLI does not infer the `@force-ui` namespace from the preset alone.

<RegistryConfigBlock />

For React, keep `style` set to `radix-force-ui` or `base-force-ui`. For the other framework registries, use the matching style value:

- `vue-force-ui`
- `svelte-force-ui`
- `ember-force-ui`

## Add components

Once the registry mapping is in place, install components with the official CLI:

```bash
npx shadcn@latest add @force-ui/button
npx shadcn@latest add @force-ui/card @force-ui/dialog
npx shadcn@latest add @force-ui/badge
```

The CLI will:

1. Fetch the registry item.
2. Resolve registry and npm dependencies.
3. Write the source files into your project.
4. Update your CSS imports and variables when needed.

## Install custom variants

Custom variants are installed with the component source.

For example, installing `@force-ui/badge` adds Force UI variants such as `warning`, `success`, `info`, `success-solid`, `warning-solid`, `info-solid`, and `error-solid`.

```tsx
import { Badge } from "@/components/ui/badge"

export function Example() {
  return (
    <div className="flex gap-2">
      <Badge variant="warning">Warning</Badge>
      <Badge variant="success">Success</Badge>
      <Badge variant="info">Info</Badge>
    </div>
  )
}
```

If you want updated variants for an existing component, reinstall that component from the Force UI registry.

## Notes for coding agents

- For a new Force UI project, run `init` with `REGISTRY_URL` set to `https://shadcn-force-ui.vercel.app/r` and a Force UI preset URL.
- For an existing project, run `apply` with the same `REGISTRY_URL` and preset URL.
- For React, use `radix-force-ui` or `base-force-ui` as the style value.
- Add the `@force-ui` registry mapping to `components.json` before using `npx shadcn@latest add @force-ui/...`.
- Theme and font changes come from `init` or `apply`. Variants come from the installed component source.

## Install the shadcn skill

Yes. Today the supported skill install is the generic shadcn skill:

```bash
npx skills add shadcn/ui
```

This gives your assistant shadcn CLI, preset, and registry workflow knowledge. It is the right install path today for assistants that support `skills`.

A dedicated Force UI skill does not exist yet. If we add one later, we can use it to encode the Force UI preset URLs, `@force-ui` registry mapping, and component-specific guidance from this page.
