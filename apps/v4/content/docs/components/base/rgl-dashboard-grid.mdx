## About

`DashboardGrid` wraps [react-grid-layout](https://github.com/react-grid-layout/react-grid-layout) with shadcn-friendly styling. Pair it with [`RGL Dashboard Tile`](/docs/components/base/rgl-dashboard-tile) for tile content.

## Usage

```tsx showLineNumbers
import { DashboardGrid } from "@/components/ui/rgl-dashboard-grid"
```

```tsx showLineNumbers
<DashboardGrid
  layout={layout}
  onLayoutChange={setLayout}
  isDraggable={editMode}
  isResizable={editMode}
>
  {layout.map((item) => (
    <div key={item.i}>{/* tile content */}</div>
  ))}
</DashboardGrid>
```

## API Reference

### DashboardGrid

Client wrapper around `WidthProvider(GridLayout)` from `react-grid-layout`.

| Prop | Type | Description |
|------|------|-------------|
| `layout` | `Layout[]` | RGL layout array; each item's `i` is the tile id |
| `onLayoutChange` | `(layout: Layout[]) => void` | Called whenever the layout updates |
| `isDraggable` | `boolean` | Enable drag — tie to edit mode in your UI |
| `isResizable` | `boolean` | Enable resize — tie to edit mode in your UI |
| `compactType` | `"vertical" \| "horizontal" \| null` | Defaults to `"vertical"` inside the wrapper |
| `className` | `string` | Extra classes forwarded to the grid container |

### DashboardDemo

| Prop | Type | Description |
|------|------|-------------|
| `embedded` | `boolean` | Tighter padding for inline docs previews |
| `initialDashboard` | `DashboardRecord` | Seed layout + widgets for all tiles; combine with `persistToStorage={false}` |
| `persistToStorage` | `boolean` | Default `true` (localStorage). `false` = session-only state that resets on refresh |

### DashboardTile / TileByWidget

Render one tile by id and widget type inside your own `DashboardGrid`.

```tsx
import { DashboardTile } from "@/components/dashboard/tile-widgets"

<DashboardTile id="tile-kpi" type="kpi" />
```

Supported `type` values: `intro` · `kpi` · `chart` · `bar` · `table` · `metric`

### DASHBOARD_EXAMPLE_SPEC

A canonical `DashboardRecord` exported from `@/lib/dashboard-example` — same shape as the live state JSON (fields: `id`, `name`, `layout`, `widgets`). Pass it directly to `DashboardDemo` for a reproducible demo state.

```tsx
import { DASHBOARD_EXAMPLE_SPEC } from "@/lib/dashboard-example"

<DashboardDemo initialDashboard={DASHBOARD_EXAMPLE_SPEC} persistToStorage={false} />
```
